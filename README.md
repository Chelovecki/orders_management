# Orders Management API

Проект представляет собой REST API для управления заказами с поддержкой аутентификации пользователей, кэширования, событийной шины и фоновой обработки. Система состоит из FastAPI приложения, PostgreSQL, Redis (кэш и брокер Celery), RabbitMQ (event bus) и Celery для фоновых задач.

---

## Технологический стек

- **FastAPI** – веб-фреймворк
- **PostgreSQL** – основная база данных
- **Redis** – кэширование заказов + брокер задач для Celery
- **RabbitMQ** – шина событий (event‑bus)
- **Celery** – выполнение фоновых задач
- **SQLAlchemy 2.0** (async) – ORM
- **Alembic** – миграции
- **Pydantic** – валидация данных
- **JWT** – аутентификация
- **Docker / Docker Compose** – контейнеризация

---

## Доступные endpoints
1. POST **/token** - создание сессии и получение токена пользователя
2. POST **/register** - регистрация пользователя
3. POST **/orders** - создание заказа
4. GET **/orders/{order_id}** - получение заказа по UUID
5. PATCH **/orders/{order_id}** - изменение заказа по UUID
6. GET **/orders/user/{user_id}** - получение всех заказов пользователя




## 🐳 Запуск проекта с Docker Compose
Перед запуском контейнеров, нужно в папке `/docker` создать файл `.env` и заполнить его по образу и подобию файла `/docker/.env.example`


Для запуска докер образов в терминале проекта перейти в директорию docker
```bash
cd docker
```

Запустить все сервисы с предварительным созданием контейнеров
```bash
docker-compose up --build
```
